FAQ

Q: Is Newt free to use?
A: Newt is an open source project, distributed under GNU Lesser General Public License.

Q: Is there anything I need to install to start using Newt?
A: No, Newt is web based and is expected to work on most major browsers with recent releases without any prior installation or set up.

Q: What is the best way to get started with Newt?
A: First, read the tutorials page and watch the associated videos.

Q: What notation is used in Newt to draw biological maps?
A: Newt supports SBGN Process Description (PD) and Activity Flow (AF) maps. Users are allowed mix up notation from these two languages at the cost of disabled validation checks during creation.

Q: What format does Newt use to save maps?
A: Newt can read SBGNML files. Models constructed with Newt can be saved in rich SBGNML, with extensions for style and annotations ("File | Save..."), or in plain SBGNML ("Export | SBGN-ML Plain"). In addition, the user can import AF models from simple, tab-delimited text files. We also support importing from and exporting to CellDesigner file format. Maps can also be exported as static (raster or vector) images.

Q: Does Newt save styling information (e.g. color, font, custom annotations, etc.)?
A: Yes, any style information per node/edge/map is saved as an extension to standard SBGNML. Thus, you can restore such information when you load such files back up on Newt but such information might be lost when imported to other tools (since style information isn't supported by standard SBGNML). If a tool is having difficulty importing our files, try exporting in plain SBGNML ("Export | SBGN-ML Plain").

Q: How can I create a reversible reaction in Newt?
A: Use of a single process for a reversible reaction is a controversial issue since it can lead to ambiguities when the process has modulators. In Newt, if a map's type is PD, it doesn't allow connecting production edges to both the input and the output ports of a process. However, one can create reversible reactions using a template ("Edit | Add from Template..."), but Newt will set the map type as "Unknown" and further editing will not go through validation checks.

Q: Can I overlay experimental data on my maps in Newt?
A: Support for this using a simple tab-delimited file is planned for a release in near future.

Q: Can I deploy my own Newt instance?
A: Yes, it is rather straightforward to deploy and maintain a local copy of Newt. In addition, Newt is specifically designed to be easily customizable (modify menu and toolbars, and other functionality) and embeddable within your own website.

Q: What if I'd like to contribute to Newt development?
A: We'd love to have more resources. Please contact us privately through this form.
